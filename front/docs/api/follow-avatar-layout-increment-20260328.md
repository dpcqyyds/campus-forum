# 关注功能UI布局增量文档（2026-03-28）

## 1. 变更目标

优化关注功能入口的视觉布局与使用习惯：

1. 帖子详情页：作者头像放在作者信息左侧区域，关注按钮放在头像下方。  
2. 公开主页：关注按钮放到账户头像下方。  
3. 个人主页「我的关注」列表：新增头像列。  

并保持逻辑：
- 打开自己的公开主页地址时，自动跳转个人主页。
- 个人不可关注自己。

## 2. 前端已完成

### 2.1 帖子详情页 `/admin/post-detail/:postId`

- 新增作者头像展示块（可点击跳转作者公开主页）
- 关注作者按钮已放到头像下方
- 自己的帖子不显示“关注作者”按钮

### 2.2 公开主页 `/admin/profile/:userId`

- 关注按钮已调整到头像下方
- 若访问的是本人ID，前端自动跳转 `/admin/profile`

### 2.3 个人主页 `/admin/profile`

- 「我的关注」表格新增头像列
- 头像和昵称均可点击进入对应公开主页

## 3. 后端接口补充要求

为提升头像展示一致性，建议补充以下字段：

## 3.1 帖子详情接口

- `GET /api/v1/posts/{postId}`
- 建议返回：`authorAvatar`（作者头像URL）

若短期无法补 `authorAvatar`，前端会使用作者首字母占位，不阻塞功能。

## 3.2 我的关注列表接口

- `GET /api/v1/profile/me/following`
- 每条记录建议包含：`avatar`

## 3.3 公开主页接口（已在前序文档约定）

- `GET /api/v1/profile/users/{userId}`
- `data.user.avatar` 保持可用

## 4. 验收项

1. 帖子详情页能看到作者头像，点击可进入该作者公开主页。
2. 帖子详情页关注按钮位于头像下方，状态切换正常。
3. 公开主页关注按钮位于头像下方，状态切换正常。
4. 打开自己的公开主页URL会跳转个人主页。
5. 个人主页“我的关注”列表可显示关注账号头像。

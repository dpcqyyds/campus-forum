# 内容发布「存为草稿」增量文档（2026-03-28）

## 1. 变更目标

在内容发布页新增「存为草稿」按钮，满足用户临时保存需求。

页面：`/admin/post-create`

## 2. 前端已完成

1. 底部操作按钮改为：
- 发布内容
- 存为草稿
- 重置

2. 点击「存为草稿」时：
- 请求仍走 `POST /api/v1/posts`
- 强制提交 `status = draft`
- 成功提示：`已存为草稿，可在“我的帖子”中继续编辑或重新发布。`

3. 草稿校验策略：
- 仅校验 `title` 与 `boardId` 必填
- 不强制校验正文完整度（便于暂存）

4. 与“我的帖子”联动：
- 草稿会出现在 `/admin/my-posts`
- 可继续编辑，或点击“重新发布”改为 `pending`

## 3. 后端接口要求

接口：`POST /api/v1/posts`

### 3.1 关键规则

1. 接收到 `status=draft` 时应按草稿保存，不应强制改写为 `pending`。
2. 草稿内容允许不完整（由后端按业务最小约束处理）。
3. 草稿作者应为当前登录用户。

### 3.2 返回结构

```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "post": {
      "id": 123,
      "status": "draft"
    }
  }
}
```

## 4. 验收项

1. 在内容发布页点击“存为草稿”可成功保存。
2. 草稿在“我的帖子”中可见，状态为 `draft`。
3. 草稿可编辑并可重新发布为 `pending`。
4. 普通发布流程不受影响（仍可提交审核/发布）。

# 我的帖子「编辑/撤回/重新发布/软删除」增量文档（2026-03-28）

## 1. 目标

规范用户在“我的帖子”页面的可操作范围：

1. 已发布（`published`）帖子：不可编辑，仅允许“删除”（软删除）。
2. 未发布通过审核的帖子（如 `draft/pending/rejected`）：可编辑。
3. `pending` 支持“撤回” -> `draft`。
4. `draft/rejected` 支持“重新发布” -> `pending`。

> 删除不做物理删除，数据库保留，状态改为 `hidden`。

## 2. 前端已完成

页面：`/admin/my-posts`

### 2.1 操作按钮规则

- `published`：显示 `删除`（危险按钮）
- `pending`：显示 `编辑` + `撤回`
- `draft`：显示 `编辑` + `重新发布`
- `rejected`：显示 `编辑` + `重新发布`
- `hidden`：不在“我的帖子”列表展示（用户侧已过滤）

### 2.2 编辑规则

- 仅 `draft/pending/rejected` 可编辑
- 编辑保存仅更新：`title/summary/content`
- 不在编辑框内开放状态选择（状态靠操作按钮流转）

### 2.3 删除确认

- 删除前弹窗确认，防误操作
- 确认后发起“软删除”状态更新

## 3. 后端接口需求

本次仍使用现有接口：

- `PATCH /api/v1/posts/{postId}`

## 3.1 需要后端支持的状态流转

1. 用户删除（软删除）：`published -> hidden`
2. 用户撤回：`pending -> draft`
3. 用户重新发布：`draft|rejected -> pending`

## 3.2 权限与约束（关键）

1. 仅帖子作者本人可操作自己的帖子流转。
2. 普通用户不允许直接将帖子改为 `published`。
3. 已发布帖子不允许用户侧编辑正文（可由后端校验拒绝）。
4. 软删除后帖子在用户“我的帖子”默认列表不可见，但管理端可见。

## 4. 建议后端错误码

- 非作者操作：`403`
- 非法状态流转：`400`
- 帖子不存在：`404`

## 5. 验收项

1. 已发布帖子在“我的帖子”无编辑按钮，仅有删除按钮。
2. 删除后用户列表不再显示该帖，管理端仍可查到（`hidden`）。
3. `pending` 点击撤回后状态变 `draft`。
4. `draft/rejected` 点击重新发布后状态变 `pending`。
5. 编辑未发布帖子后，管理端查看内容同步更新。

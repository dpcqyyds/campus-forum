# 前置审核增量文档（2026-03-21）

## 1. 目标

在“用户发帖 -> 管理员审核”之间新增前置审核，减少审核员人工工作量。

## 2. 前端已完成改动

页面：`/admin/post-create`

1. 发布请求失败时，前端可识别后端前置审核拦截错误。
2. 当后端返回 `422` 或业务码 `42201` 时，页面展示“前置审核提示”面板。
3. 提示面板展示：
- `message`
- `riskLevel`
- `hitWords[]`
4. 普通错误仍沿用原错误提示逻辑。

## 3. 后端接口契约（新增约定）

接口：`POST /api/v1/posts`

### 3.1 未命中前置规则
- 正常入库
- 继续走原审核流程（建议仍写 `status=pending`）

### 3.2 命中前置规则（拦截）
- HTTP 状态：建议 `422`
- 响应：

```json
{
  "code": 42201,
  "message": "内容命中违禁规则",
  "data": {
    "message": "请修改后再提交",
    "riskLevel": "high",
    "hitWords": ["示例词1", "示例词2"]
  }
}
```

## 4. 检测字段范围（建议）

- `title`
- `summary`
- `content`
- `tags`
- `linkTitle`
- `linkSummary`

## 5. 风险等级建议

- `low`：弱风险，可仅记录
- `medium`：建议提示并进入待审
- `high`：建议拦截提交（即本次 422）

## 6. 审核台联动建议

即使前置审核已存在，管理员审核仍保留。
- 审核列表建议支持 `riskLevel` 排序：`high > medium > low`
- 支持按 `riskLevel` 筛选

## 7. 验收用例

1. 正常帖子发布成功，状态进入待审核。
2. 帖子命中违禁词，发布失败并看到前置审核提示面板。
3. 提示面板可看到命中词与风险等级。
4. 非前置审核错误（如网络异常）仍显示原错误提示。

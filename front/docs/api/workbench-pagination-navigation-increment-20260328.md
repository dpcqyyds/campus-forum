# 工作台分页与全量浏览增量文档（2026-03-28）

## 1. 变更目标

解决工作台“帖子数量太少、无法持续浏览”的问题：
- 支持分页浏览全部已发布帖子
- 支持每页条数切换（10/20/50/100）
- 与现有推荐/热门/最新及检索筛选并存

页面：`/admin/home`

## 2. 前端已完成

1. 列表改为分页模式，显示：
- 当前页 / 总页数
- 总条数
- 上一页 / 下一页按钮
2. 新增每页条数选择：
- `10 / 20 / 50 / 100`
3. 保持原有功能不变：
- 推荐/热门/最新
- 全文检索与高级筛选
4. 从列表进入帖子详情时，会携带 `page/pageSize/feed/filters`，返回列表可恢复上下文。

## 3. 后端接口要求

接口：`GET /api/v1/posts/published`

### 3.1 查询参数

- `page`：页码，从 1 开始
- `pageSize`：每页条数（建议上限 100）
- 其余参数保持：`keyword,author,tag,boardId,dateFrom,dateTo,feed`

### 3.2 返回结构（必须）

```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "list": [],
    "total": 128,
    "page": 2,
    "pageSize": 20
  }
}
```

## 4. 后端注意事项

1. 必须按 `page/pageSize` 真正分页，不要始终固定返回前 10 条。
2. `total` 必须是符合当前筛选条件的总数，不是当前页数量。
3. 当 `page` 超过总页数时，建议返回空 `list` 与原始 `total`。

## 5. 验收项

1. 工作台可连续翻页浏览全部已发布帖子。
2. 切换每页条数后，列表和总页数正确变化。
3. 应用筛选后分页仍正确（`total/page/pageSize` 一致）。
4. 进入详情再返回，页码与筛选状态可恢复。

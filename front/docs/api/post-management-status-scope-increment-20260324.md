# 帖子精细化管理「仅展示已发布/已下架」增量文档（2026-03-24）

## 1. 变更目标

在超级管理员/教师使用的“帖子精细化管理”页面中，帖子列表默认只展示：
- 已发布（published）
- 已下架（hidden）

不再展示草稿/待审核/驳回等状态。

## 2. 前端已完成

页面：`/admin/posts`

1. 状态筛选默认值改为：`published,hidden`
2. 状态下拉仅保留：
- 已发布 + 已下架
- 已发布
- 已下架
3. 前端在拿到列表后，二次过滤仅保留允许状态（兜底逻辑）

## 3. 后端建议增量（推荐）

接口：`GET /api/v1/posts`

### 3.1 建议支持状态多选参数
- 查询参数：`status=published,hidden`
- 含义：按逗号分隔进行 IN 查询

### 3.2 兼容口径
- 若传单值：`status=published` -> 等值过滤
- 若传多值：`status=published,hidden` -> IN 过滤
- 若不传：按原逻辑

> 当前前端已做本地过滤兜底，但推荐后端直接过滤，减少不必要数据传输。

## 4. 验收项

1. 进入 `/admin/posts` 默认只看到 published/hidden 状态帖子。
2. 切换状态筛选为“已发布”仅显示 published。
3. 切换状态筛选为“已下架”仅显示 hidden。
4. 不出现 draft/pending/rejected 状态帖子。
